## 1. Skills instaladas no workspace

### Starter / Jornada

- `onboarding-checklist` — conduz a jornada principal do Starter Kit.
- `continuar-jornada` — retoma fluxo de onboarding quando necessário.
- `primeira-vitoria` — cria o primeiro artefato real no workspace.
- `gera-log-jornada` — gera log/feedback estruturado da jornada.
- `wizard-agente` — configura identidade/persona do agente.
- `wizard-aluno` — configura dados e preferências do usuário.
- `wizard-autonomia` — configura autonomia operacional.
- `wizard-workspace` — organiza estrutura do workspace.
- `wizard-conectar` — conecta GitHub, Search API e browser.
- `wizard-whisper-quick` — configura áudio/Whisper.

### Operacional

- `backup-workspace-github` — backup automático do workspace no GitHub.
- `commit-diario-workspace` — rotina de commit diário do workspace.
- `cron-resume-wizards` — retomada automática de wizards pausados.
- `seguranca-checklist` — checklist/auditoria de segurança.

### Planejamento

- `brainstorming` — ideação estruturada.
- `writing-plans` — criação de planos.
- `executing-plans` — execução de planos.
- `verification-before-completion` — verificação antes de marcar tarefa como concluída.

### Canais

- `wizard-whatsapp` — configuração/uso de WhatsApp.

### Perícias

- `pericia-psiquiatrica` — kit de perícias psiquiátricas.

Inclui referências internas para:

- perícia trabalhista;
- perícia cível/capacidade civil;
- perícia administrativa/funcional;
- estrutura de laudo;
- respostas a quesitos;
- prompts seguros;
- checklist de anonimização.

## 3. Recomendação prática

Não instalar tudo. Manter o workspace enxuto.

Prioridade atual recomendada:

1. Consolidar `pericia-psiquiatrica`.
2. Criar templates específicos de perícias conforme uso real.
3. Usar `skill-creator` quando formos evoluir skills.
4. Usar `healthcheck` quando houver problema técnico.
5. Usar `clawhub` apenas quando quisermos buscar skills novas.

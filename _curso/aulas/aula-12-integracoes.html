Aprofundamento do Bloco 1

Falar não é operar.

Cloud agent fala. ChatGPT puro responde quando você chama, dá conselho, escreve texto. Agente conectado é outra categoria — ele lê seu calendar real, escreve no seu Notion real, abre PR no seu GitHub real, classifica seu inbox real.

Esse é o momento em que aluno PME para de pensar "tenho um chatbot melhorado" e entende "tenho um funcionário digital integrado". A diferença não é técnica — é de papel.

Realidade do PME hoje

✓ Agente atualiza CRM sem você abrir o painel

✓ Agente lê inbox e classifica os 5 emails que precisam de você hoje

✓ Agente abre ticket no Linear quando aluno reclama em comunidade

✓ Agente confere agenda e te avisa do próximo evento + briefing dos participantes

✓ Agente edita roadmap no Notion baseado em decisões do dia

✓ Agente commit + push + PR diário do workspace pra GitHub privado

Tese central: "Toda ferramenta moderna tem API. Você dá uma credencial pro agente, ele opera. Não tem 'integração mágica' — tem 4 padrões de auth que cobrem 90% dos casos."

Aprofundamento do Bloco 2 · framework

4 padrões de auth.

Esse é o coração da aula. Aprende esses 4 e você sabe conectar QUALQUER ferramenta moderna — não importa se já existe, se é nova, se é interna da sua empresa. A doc oficial de cada ferramenta diz qual padrão ela usa. O agente lê, conduz, valida.

# Padrão Como funciona Exemplos
1 Token simples (API key) Gera no painel da ferramenta · cola no .env (known key) · agente usa via Bearer Notion · GitHub · Linear · Stripe · Buffer · Anthropic · Brave
2 OAuth flow Fluxo de browser · agente recebe access_token · refresha quando expira Slack (como ferramenta de busca) · Twitter/X API · Spotify · Hubspot
3 CLI dedicada Instala CLI da plataforma · autentica uma vez via browser · agente roda comandos Google Workspace: gog CLI (Gmail/Calendar/Drive/Docs/Sheets) · AWS: aws · Azure: az
4 IMAP/SMTP + skill Protocolo padrão · skill custom abstrai · útil pra email Gmail (alternativa ao gog) · Outlook · Yahoo · servidores email custom

Princípio: "Você decora 4 padrões. Não decora 50 APIs." E mais importante: "a doc oficial de cada ferramenta é a fonte da verdade. O agente lê, conduz o setup, valida. Você só fornece a credencial quando ele pedir."

Por que não MCP? MCP servers são uma alternativa real, mas o curso V2 decidiu por API direta + 4 padrões de auth. Razão: 4 padrões cobrem 90% dos casos e ensinam modelo mental transferível. MCP exige setup adicional por servidor e maturidade que aluno PME não tem ainda. Bônus técnico cobre MCP pra quem quiser depois.

A-ha #8 acontecendo: ele NÃO TÁ DESCREVENDO um Notion. Ele TÁ NO MEU Notion. Diferença categorial — não é mais conselho. É operação real em cima de dado real.

Pegadinha do Notion · share com integração

O passo que muita gente esquece: token criado, configurado, salvo — mas o agente não vê nada. Razão: Notion exige que você vá em cada page ou database que quer compartilhar com a integração, clique em "Add connection", escolha sua integração na lista. Sem isso, integração tem permissão zero. Não é bug — é design de privacidade do Notion.

Token simples é o caminho mais comum. Notion · GitHub · Linear · Stripe · Buffer — todos seguem mesmo padrão. Cheatsheet do kit tem prompt pronto pra cada um — você cola e ele conduz.

Prompt pronto · conectar GitHub via PAT

Prompt #2 · conectar GitHub cola no Telegram do agente
Quero conectar meu GitHub. Vai na doc oficial em https://docs.openclaw.ai/cli/secrets e em https://docs.github.com/en/rest, me explica: 1. Onde gerar Personal Access Token (PAT) — fine-grained ou classic 2. Qual scope mínimo (recap A10): repo se quero commit/push, apenas read se só vou ler 3. Tempo de expiração (30/60/90 dias) e como rotacionar 4. Adicionar como GITHUB_TOKEN no .env (known key OpenClaw) 5. Validar com leitura de 1 repo de teste Me pede o token quando pronto. Cron mensal pra me lembrar de rotacionar 7 dias antes do vencimento (gancho A9).
💡 PAT expira em 30/60/90 dias por padrão (boas práticas GitHub recomendam). Cron mensal te lembra antes — não descobre vencido em meio a uma operação importante.

Comando natural pra criar a skill commit-diario-workspace

Prompt · criar skill commit-diario cola no Telegram do agente
Cria skill `commit-diario-workspace` que: - cd no workspace - git add . - git commit com mensagem "Auto: snapshot YYYY-MM-DD HH:mm" - git push origin main - me avisa no tópico Operação Roda toda noite 23h59 BRT em sessão isolada. Skip silencioso se git status limpo (nada pra commitar). Pausa skill se token expirar — me avisa pra rotacionar.
💡 Skill rascunhada já tá no kit em starter-kit/skills/operacional/commit-diario-workspace/. Aurora não escreve do zero ao vivo — confirma criação, agenda cron, primeira execução em < 30s.

Princípio do bloco: skill custom = automação específica que combina API + cron + notificação. Você não escreve código — agente escreve. backup-workspace-github (A2) faz snapshot. commit-diario-workspace (A12) dá granularidade diária. Os dois juntos fecham operação.

Recap A10/A12 sobre rotação: PAT expira. Cron mensal (gancho A9 — meta-cron) lembra de rotacionar 7 dias antes do vencimento. Você não descobre token vencido em emergência — você rotaciona com calma uma vez por mês.

Aprofundamento do Bloco 5 · vitória visível #2

Google Workspace via gog CLI.

Google é diferente — caminho oficial é gog CLI (não gcloud). gog cobre Gmail + Calendar + Drive + Docs + Sheets + Contacts num CLI unificado. Boas notícias: já vem instalado no OpenClaw 2026.4+. Setup é 1 fluxo OAuth pra tudo.

Comandos exatos

Terminal Managed setup gog em 4 comandos
$gog --version # confere se já tá instalado (vem no OpenClaw 2026.4+) $gog auth credentials /path/to/client_secret.json # aponta pro client_secret.json do Google Cloud Console $gog auth add seu@email.com --services gmail,calendar,drive,contacts,docs,sheets # abre fluxo OAuth no browser · loga uma vez · cobre tudo $gog auth list # confirma contas autenticadas + serviços ativos
▶ Setup OAuth roda 1x. Daí em diante, agente acessa Gmail/Calendar/Drive sem fricção.

Uso típico

Terminal Managed comandos gog mais comuns
$gog calendar events <calendar-id> --from 2026-01-01 --to 2026-01-07 # lista eventos no range $gog gmail search 'newer_than:7d' # busca emails dos últimos 7 dias $gog drive search "minha pasta" # busca arquivos no Drive
▶ Sintaxe consistente entre serviços. Cheatsheet completo: github.com/steipete/gogcli

Aprofundamento do Bloco 6

Browser control: quando não tem API decente.

Cenário típico do PME brasileiro: sistema legado interno (ERP antigo, CRM custom da empresa), SaaS pequeno sem API pública nem MCP, painel administrativo só funciona via UI. Solução: agente abre navegador, faz login, navega, lê, atualiza.

Comando natural · ler painel CRM legado

Prompt · browser control cola no Telegram do agente
Abre o painel do meu CRM em [URL]. Login: [email]. Senha: lê do 1Password vault `OpenClaw/CRM` (gancho A10). Acessa a tela de leads, lista os 5 com status "aguardando contato", me manda no Telegram tópico Suporte com nome + valor + telefone. Se aparecer 2FA, PARA e me avisa — não tenta burlar. Se sessão expirar no meio, reconecta sob demanda.
💡 Browser control é caminho legítimo. "Mesma coisa que você faz manualmente, só que automatizado." Sem hack, sem gambiarra.

Ressalvas (recap A10)

✓ Scope de estagiário: agente NÃO tem write em sistema crítico sem confirmação humana (recap A5/A10)

✓ 2FA bloqueia: se o site exige 2FA, agente PARA e te avisa — não tenta burlar. Mover pra conta de serviço sem 2FA ou usar app password se a plataforma oferece

✓ Sessão expira: agente reconecta sob demanda — se quebrar a sessão, te avisa e pede pra reautenticar

✓ Anti-bot pode pegar: sistemas com Cloudflare/Distil/captcha podem bloquear. Browser control resolve UI normal — não é solução pra anti-bot avançado (vai pra bônus técnico R3)

Doc oficial: docs.openclaw.ai/tools/browser + browser-control — comandos avançados, headless vs headed, captura de screenshot, debug.

Aprofundamento do Bloco 7

Webhooks pra eventos + defesa email.

Webhook = agente REAGE a evento externo. Não é só responder quando você chama — é processar coisa que aconteceu sem você precisar avisar.

Exemplos PME

✓ Ticket Linear urgente → notifica tópico Suporte

✓ Pagamento Stripe > R$ 5k → notifica + cria card no Notion

✓ PR mergeado no GitHub → roda análise + posta resumo no Slack

✓ Evento Calendar começando em 15min → manda lembrete + briefing dos participantes

✓ Cliente reclamou em comunidade → cria ticket Linear automático com label "voz do cliente"

Email é caso crítico · vetor #1 de prompt injection

Recap A10 Bloco 5: agente lê inbox, classifica, gera digest matinal. Defesa anti-injection no SOUL é OBRIGATÓRIA. Sem o bloco "Defesa contra input externo" no SOUL, agente que processa email é vulnerável a qualquer instrução escondida que chegar via remetente convincente.

Caso N8n/Zapier · arquitetura externa

Se você tem fluxo externo escutando WhatsApp e disparando agente via webhook (caso real do Bruno cobrindo openclawzinho), agente trata o input como externo — defesas A10 sempre. Não importa que o webhook veio do "seu" N8n — pra fins de prompt injection, é input não-confiável.

Doc oficial: docs.openclaw.ai/cli/webhooks + plugins/webhooks — setup técnico de webhook receiver, autenticação, signature verification.

Troubleshooting

Travou? Antes de pedir ajuda.

Os 7 erros mais comuns ao integrar ferramentas e o caminho exato pra resolver. Se passou pelos 7 e ainda não saiu do lugar, joga screenshot/log no grupo Telegram.

Erro / sintoma Causa provável Caminho
Notion API retorna 404 em page que existe Page não foi compartilhada com a integração — Notion exige isso explícito Vai na page no Notion · "..." → "Add connection" → escolhe sua integração na lista. Repete pra cada page que o agente precisa ler/escrever
GitHub PAT funcionou ontem, hoje retorna 401 PAT expirou (default 30/60/90 dias) ou token foi revogado por política do repo Vai em github.com/settings/tokens · gera novo PAT com mesmo scope · atualiza GITHUB_TOKEN no .env · roda openclaw secrets reload
gog --version retorna "command not found" Versão OpenClaw < 2026.4 (gog não veio instalado por padrão) Atualiza OpenClaw via painel Hostinger ou pede pro agente instalar gog manual via fluxo do gogcli README
OAuth do gog falha com "invalid_client" client_secret.json baixado da conta Google errada · ou projeto Google Cloud sem OAuth ativado Confere se o client_secret.json é da conta certa · habilita "Gmail API · Calendar API · Drive API" no Google Cloud Console · gera credentials OAuth tipo "Desktop app"
Browser control para no 2FA do CRM Site exige confirmação por SMS/app · agente não tem como interceptar Cria conta de serviço dedicada SEM 2FA pra essa integração específica (recap A5 — funcionário digital com login próprio). Ou usa app password se a plataforma oferece
Webhook recebe payload mas agente não processa Signature verification falhando · ou skill receiver não tá registrada Confere assinatura do webhook (cada plataforma tem header próprio — X-Hub-Signature no GitHub, Stripe-Signature no Stripe). Roda openclaw plugins list pra ver se webhook receiver tá ativo
Skill commit-diario-workspace não roda às 23h59 Cron não foi agendado, ou timezone do Managed difere de BRT, ou git push falhou silencioso openclaw crons list · confere se cron tá ativo · confere timezone (date no terminal) · roda skill manual pra ver erro real (sem skip silencioso)

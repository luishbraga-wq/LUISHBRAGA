Pre-flight check

Antes de apertar play.

A1 é sequencial — não dá pra pular passo. Reserva 30-45 minutos sem interrupção (idealmente A1 + A2 em sequência) e tem em mãos:

✓ cartão de crédito (pra OpenAI API)

✓ conta ChatGPT Plus criada e logada (recomendado, não obrigatório no day 1)

✓ navegador atualizado

✓ celular com Telegram instalado

✓ link da comunidade aberto (t.me/cursoopenclaw) pra travar em algum passo

Pegadinha clássica: aluno PME confunde platform.openai.com (onde você gera API key) com chatgpt.com (onde você assina ChatGPT Plus). SÃO SITES DIFERENTES. Se você cadastrou cartão no chatgpt.com pensando que era pra API, abriu assinatura ChatGPT Plus — não API. Cancela e vai pro platform.openai.com.

Bloco 3 · Fase A

Configurar credencial OpenAI API.

Esses 5 passos são todos clicáveis no painel — sem terminal, sem comando. Pega ~5 minutos se você for com calma.

1

Criar conta no platform.openai.com

Abre platform.openai.com (NÃO confundir com chatgpt.com). Cria conta com email pessoal ou Google. Pula questionário "what brings you to OpenAI?" — qualquer resposta serve.

2

Cadastrar cartão + comprar crédito (US$ 5-10)

Settings → Billing → Add payment method (cartão de crédito internacional) → Add credit balance (US$ 5-10). Pagamento processa em ~30 segundos.

3

Gerar API key

API Keys → + Create new secret key. Nome: openclaw-managed. Permissions: All. Copia a chave — começa com sk-proj-.... Cola num lugar seguro temporariamente (vai usar no Passo 5).

4

Contratar Hostinger Managed OpenClaw

Abre hostg.xyz/SHJMI · aplica cupom BRUNOOKAMOTO · finaliza · aguarda provisionamento (3-5 min). Email da Hostinger chega quando OpenClaw estiver pronto.

5

Logar painel OpenClaw com API key

Email Hostinger → link de acesso → painel OpenClaw → setup wizard pede API key → cola a chave do Passo 3. Painel ativa · agente já consegue responder.

Atenção: a API key só aparece UMA VEZ. Se fechar a janela sem copiar, precisa gerar outra. Cola num lugar seguro AGORA — bloco de notas, password manager, Slack do you mesmo. Princípio A10 (segurança): NÃO compartilha · NÃO commita em repo público.

Por que US$ 5-10? Cobre meses de uso PME normal (Whisper + embeddings + algumas chamadas DALL-E). Se acabar, você só recarrega quando o saldo cair — sem assinatura mensal automática.

Caiu na landing errada? Se o site oferecer VPS standalone em vez de Managed OpenClaw, NÃO CANCELA — VPS é útil pra outras coisas depois (rodar outros OpenClaw, Claude direto, MCPs). Mas pra esse curso você precisa especificamente do Managed OpenClaw. Pega o Managed pelo link acima também. Se errou e quer reembolso do VPS: chat com agente Hostinger no painel deles.

Validação Fase A: agente respondendo no Gateway do Managed OpenClaw via API key. Manda oi, quem é vc? e ele responde. Se respondeu, Fase A OK — pronto pra Fase B (OAuth no terminal).

Bloco 3 · Fase B

Ativar OAuth ChatGPT — segundo passo crítico.

Aqui você abre o terminal Managed pela primeira (e provavelmente única) vez no curso. No V2, esse é o ÚNICO comando que você cola manualmente — daqui em diante o agente faz tudo via Telegram.

Por que esse comando é manual

Comandos de elevação de privilégio (mudar autenticação do agente) JAMAIS passam pelo próprio agente — é princípio defensivo do OpenClaw. Você decide quando ativar OAuth. Aprofunda na A10 (segurança operacional).

Passo 1 — Abrir terminal Managed

  1. Painel Hostinger → menu OpenClaw → seu OpenClaw
  2. Botão "Abrir linha de comando (CLI)" abaixo do nome do seu OpenClaw
  3. Abre janela com terminal preto · aguarda 2-3s até aparecer o prompt ($)

Aluno PME nunca usou terminal? Tudo bem — esse é o único comando manual que você cola no day 1. Esse terminal não é Linux completo · é uma versão restrita (lshell) que só aceita comandos openclaw *. Não tem como você quebrar nada com comando errado — o sistema bloqueia tudo que não é OpenClaw.

Passo 2 — Rodar comando OAuth

Cola o comando abaixo no terminal e dá Enter:

Terminal Managed Hostinger cola e dá Enter
$openclaw models auth login --provider openai-codex --set-default
▶ Resultado: terminal mostra URL de OAuth · você abre no navegador · loga com sua conta ChatGPT Plus (a mesma da assinatura) · autoriza · volta pro terminal · OpenClaw confirma sucesso. Fluxo de ~30 segundos.

Passo 3 — Validar Fase B

Terminal Managed validação
$openclaw models auth list # mostra providers configurados + qual é default # esperado: openai-codex marcado como [default] $openclaw status # confirma que codex provider está ativo + agente saudável

Resultado: agente passa a usar GPT 5.5/5.4 da sua assinatura ChatGPT Plus via OpenAI Codex (mais barato + melhor pra agentic). API key da Fase A fica como rede de segurança automática — se OAuth falhar, agente cai nela sem você precisar fazer nada.

Conceito-chave

Por que 2 credenciais — API key + OAuth.

Aluno frequentemente pergunta: "se já tenho API key, pra que OAuth?". Resposta: cada uma cobre coisas diferentes. Não substituem — complementam.

Credencial O que cobre Custo
API key (Fase A) Whisper · embeddings · memória semântica · DALL-E · fallback automático Crédito pré-pago US$ 5-10
OAuth ChatGPT (Fase B) GPT 5.5/5.4 pra inferência · raciocínio · agentic tasks Já incluso na assinatura (R$ 100/mês)

Frase-mãe da A1: API key autentica seu OpenClaw e libera Whisper + embeddings. OAuth dá acesso ao GPT 5.5 da sua subscription. Você precisa das duas — uma não substitui a outra.

Sem ChatGPT Plus? Pula Fase B · agente roda só com API key da Fase A · funciona, mas paga TUDO via crédito pré-pago (mais caro pra agentic). Recomendação: assina ChatGPT Plus (R$ 100/mês) e faz Fase B — economia significativa em uso intenso.

Bloco 4 · A-ha #1

Agente responde + se conhece.

Com Fase A + Fase B prontas, você pode falar com o agente. Esse é o momento mais forte da A1 — não dilui em cliques, é onde aluno PME para de pensar "comprei um curso" e começa a pensar "tenho um agente".

Primeira mensagem

Pelo painel Gateway (chat web do OpenClaw), manda:

Prompt #1 · primeira mensagem cola no Gateway
oi, quem é vc?
💡 Esperado: agente responde quem é (genérico no day 1 — vai ganhar personalidade na A5). Se respondeu, Fase A + Fase B OK.

Onboarding — agente cria auto-conhecimento

Cola este prompt em seguida — agente vai ler a documentação oficial do OpenClaw e gerar MEMORY.md com auto-conhecimento estruturado. Agente passa a saber o que é, o que faz e o que não faz.

Prompt #2 · onboarding canônico cola no Gateway · primeira mensagem séria
Leia https://docs.openclaw.ai/llms-full.txt Em português simples me explique: 1. O que vc é e como funciona 2. O que consegue fazer hoje (canais, ferramentas, integrações) 3. O que NÃO consegue fazer (limitações reais) 4. Comandos principais que eu posso usar pra te configurar 5. Onde guarda memórias, skills, configurações e logs Salve esse resumo em MEMORY.md como "auto-conhecimento do agente" e me mostre o resultado.
💡 Resultado: agente lê doc · processa · cria MEMORY.md · retorna resumo. A-ha #1 conquistado: "meu agente respondeu + se conhece".

Por que esse prompt funciona: ele cobre os 4 elementos que o agente precisa pra ser útil: identidade ("o que sou"), capacidades ("o que faço"), limitações ("o que NÃO faço"), navegação ("como você me opera"). Salvar em MEMORY.md garante que o agente consulta isso sempre — não esquece. Princípio do curso: "seu agente lê documentação".

Bloco 5 · Telegram pro Quick Win

Agente no seu celular.

Antes de fechar a aula, mais 1 setup essencial: conectar o Telegram. Razão: o chat do painel Hostinger (Gateway) não aceita upload de arquivo. E na próxima aula sequencial (A4), você vai precisar enviar arquivo pro seu agente — o Starter Kit em zip. Telegram resolve isso. Setup leva 2 minutos.

Passo 1 — Criar bot no @BotFather

  1. Abre @BotFather no Telegram
  2. Digita /newbot
  3. Escolhe nome amigável (ex: "Meu Agente")
  4. Escolhe handle único terminado em _bot (ex: meu_agente_2026_bot)
  5. BotFather retorna o token (formato 123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11) · copia

Token = senha do bot. Não compartilha em chat público · não commita em GitHub · se vazar, no @BotFather: /revoke → gera novo token → atualiza no agente. Princípio A10.

Passo 1.5 — Disable Privacy (CRÍTICO pra grupo)

Por default, o bot só recebe mensagens onde é mencionado (@meu_agente_bot) ou comandos (/start). Em DM 1-1 isso não importa. Mas no momento que você for colocar o bot num grupo da equipe ou supergrupo com tópicos (A4 modular cobre), ele vai ficar mudo — não consegue ler conversa pra entrar em contexto, monitorar, ou aplicar allowlist por contexto.

Solução: desabilitar privacy mode AGORA, no @BotFather, antes de adicionar o bot em qualquer grupo. Faz parte do setup mínimo.

Telegram · @BotFather conversa direta com o BotFather
/setprivacy # BotFather pede pra escolher qual bot # escolhe o bot que você acabou de criar # BotFather mostra status atual: 'Enabled' (default) # clica em [Disable] # confirmação: 'Privacy mode is disabled for <bot>.'
▶ Resultado: bot passa a receber TODAS as mensagens dos grupos onde for adicionado (não só @menções e comandos). Sem isso, agente em grupo é cego.

Bot já tá num grupo? Privacy só aplica a partir da próxima vez que o bot for adicionado. Se mudou privacy DEPOIS de já ter colocado o bot no grupo: remove o bot do grupo + adiciona de novo. Aí privacy nova passa a valer.

Por que isso é setup, não tweak avançado: Telegram em DM 1-1 sempre funciona (privacy não afeta DM). Mas grupo é o canal mais útil pra agente PME — equipe inteira marca o agente, ele lê contexto, responde no canal certo. Sem privacy disabled, esse caso de uso morre. Faz agora pra não esquecer depois.

Passo 2 — Conectar bot ao agente

No terminal Managed, cola:

Terminal Managed substitui <token> pelo token do BotFather
$openclaw channels add --channel telegram --token <token-do-botfather> # exemplo: $openclaw channels add --channel telegram --token 123456:ABC-DEF... # validar: $openclaw channels list
▶ Resultado: telegram aparece como conectado em channels list.

Passo 3 — Testar no celular

  1. Abre teu novo bot no Telegram (busca pelo handle que você escolheu)
  2. Manda /start ou qualquer mensagem
  3. Agente responde no Telegram em segundos

Vitória visível: agente respondendo no celular do aluno (não só no terminal/painel). Saiu da web pro mobile sem fricção. Quick Win conquistado — pronto pra A3 (Starter Kit).

A aula A4 (modular) vai aprofundar Telegram (tópicos por tema · grupo da equipe · allowlist por comando natural · ressalva sobre WhatsApp). Aqui foi só DM funcional — o suficiente pra A3 jogar o zip do Starter Kit.
